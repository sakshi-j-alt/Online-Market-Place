Our Journey of building Nearbuy NOW

This website was made as a project by a group of 4 Second Year Engineering Students.

The objective of the “Project Based Learning” activity given by our college was to solve a real problem and learn through that process.

We set out to build something unique that was outside our skill set and acquire the skill by building the project. After some brainstorming we decided to make this project of Online MarketPlace for Local Vendors. This website would give the local vendors and consumers an online platform to sell and buy products respectively.

Chat Gpt: Our fifth partner in this project

Let’s face it, the age of Ai is here. Either adapt with it or be replaced by it.

I assume that most of the people reading this have used a Chat-based AI. You enter a prompt and AI answers your prompt, then you type in another prompt. This is like having a text conversation with AI.

As of now, ChatGpt isn’t clever enough to generate a whole complex website with a single prompt. So it's clear AI won’t replace the web developer's job as of now.

So how do you use it to build a whole website?

Spoiler: You still have to do the hard work.

How about going prompt by prompt and developing code - snippet by snippet.

We were clear about building an “Online Marketplace Website for Local Vendors” but didn’t know how to go about the project. All we knew about web development was frontend HTML and CSS.

We didn’t even have a solid idea of Javascript.

With a rough idea in mind, we started making the HTML and CSS of the website. ChatGpt isn’t efficient in making a whole web page, but how about prompting it to code a section of the site at a time and then improving the code by iteration. That's what we did.

Event 1: We enter the prompt

Event 2: AI generated the code.

Event 3: We understood the code (most important).

Event 4: We didn’t like how part of the code worked.

Event 5: We prompt AI to make changes.

Event 6: AI gives the updated code.

The cycle continues till we get satisfied by the result. While the process may seem linear, one has to put in effort to understand the code, make the changes yourself, see if it works, why it doesn't work, and so on. Many times the code given by AI simply doesn’t work.

A nuance here is that one still has to have a good overview of how the code works. A huge relief for me was that I no longer had to do the tedious work of memorizing syntax and worry about little errors.

Our confidence in AI got stronger after we started seeing the results. We trusted it with teaching us JavaScript and the backend Node.js and database management system. We went on to add functionality one by one with a bit of skeptism to finally build this interconnected website.

Although the functionality of this website is limited, we went from being totally clueless about how a website is built to finally building this website with JavaScript and backend logic.

Feel free to play around with our project.
